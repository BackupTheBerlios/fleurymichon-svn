\chapter{Le serveur Fleury}

\section{Installation}

Pour installer Fleury il suffit d'entrer les commandes suivantes :
\\\\
Pour creer un nouveau repertoire : \\
\verb+mkdir fleury+ \\\\
Pour deplacer la tarball dans le repertoire : \\
\verb+mv fleury-1.0.tgz fleury+ \\\\
Pour entrer dans le repertoire : \\
\verb+cd fleury+ \\\\
Pour extraire la tarball : \\
\verb+tar -xzvf fleury-1.0.tgz+ \\\\
Pour compiler sous Linux et systemes d'exploitation compatibles : \\
\verb+./makefleury.sh+ \\\\
Pour compiler sous Cygwin : \\
\verb+./cygfleury.sh+ \\\\
Pour executer Fleury : \\
\verb+./fleury+ \\\\
Pour quitter Fleury : \\
\verb+killall -s 3 fleury+

\chapter{Le client Michon}

Michon est un client IRC fait selon les normes IRC fixees par la RFC. Ainsi donc il ne devrait poser aucun problemes a communiquer avec tous serveurs irc que l'on pourra trouver faits a cette norme. Les paragraphes suivants vous permettront de vous familiariser avec l'utilisation de ce client.

\section{Installation}

L'installation du client Michon est exclusivement realisee apres compilation depuis les fichiers sources du programme, disponibles en format compresse dans une tarball. Cependant la compilation depend des bibliotheques de developpement standard et Qt 3.
\\\\
Pour installer Michon il suffit d'entrer les commandes suivantes :
\\\\
Pour creer un nouveau repertoire : \\
\verb+mkdir michon+ \\\\
Pour deplacer la tarball dans le repertoire : \\
\verb+mv michon-1.0.tgz michon+ \\\\
Pour entrer dans le repertoire : \\
\verb+cd michon+ \\\\
Pour extraire la tarball : \\
\verb+tar -xzvf michon-1.0.tgz+ \\\\
Pour compiler : \\
\verb+qmake && make+ \\\\
Pour executer Michon : \\
\verb+./michon+

\section{Utilisation}

Afin d'utiliser IRC, l'utilisateur du client doit faire savoir ce qu'il souhaite faire a travers des commandes comprehensibles par le serveur et le client. Les commandes utilisables en question sont enumerees par la suite afin d'indiquer ce qui vous est disponible et comment utiliser le client.

\subsubsection{La commande \emph{connect}}

C'est elle qui va établir une connexion avec le serveur (par défaut \emph{localhost} sur le port 6667), en lui envoyant la combinaison de 
requêtes NICK, USER et PASS, dont les paramètres seront définis par l'utilisateur dans l'onglet \emph{Settings}.
\\\\
\textit{Syntaxe : /connect <adresse de l'hôte> (<port>)}

\subsubsection{La commande \emph{join}}

Lorsque l'utilisateur désira créer ou rejoindre un canal, il n'aura qu'à utiliser la commande \emph{join}, en lui spécifiant en argument le nom du canal 
en question (précédé d'un dièse).
\\\\
\textit{Syntaxe : /join <canal>}

\subsubsection{La commande \emph{msg}}

La commande \emph{msg} sera utlisée lorsque l'on désirera entrer en communication avec un utilisateur en particulier ou un canal tout entier. De plus, lorsque l'onglet selectionne est un canal, la validation d'un texte ne commencant pas par un slash en fera l'envoi vers le canal actif.
\\\\
\textit{Syntaxe : /msg <nick ou canal> <texte à envoyer>}

\subsubsection{La commande \emph{part}}

L'utilisateur utilisera la commande \emph{part} lorsqu'il désirera quitter ou détruire un canal.
\\\\
\textit{Syntaxe : /part <canal>}

\subsubsection{La commande \emph{kick}}

Si le modérateur du canal n'est pas très enjoué par le comportement de certains utlisateurs, il n'aura qu'à effectuer une commande \emph{kick} pour l'éjecter
violemment du dit-canal (à utiliser avec parcimonie sinon cela peut entraîner une forte dépendance voir la mort).
\\\\
\textit{Syntaxe : /kick <canal> <nick> (<raison>)}

\subsubsection{La commande \emph{quit}}

L'utilisateur se servira de la commande \emph{quit} s'il désire quitter le serveur courant.
\\\\
\textit{Syntaxe : /quit <raison>}
